# AI Workspace Assistant

A small, professional, dependency-free Python CLI that **analyzes a software project
and generates a useful Markdown report** — what languages are present, how many files
of each type, and how much space they take up.

> Built with modern Python (3.13+), `argparse`, full type hints, [Ruff](https://docs.astral.sh/ruff/)
> linting/formatting, and `pytest`. Designed to be easy to extend.

---

## Features

- 🔍 **Scans** any project directory (recursively).
- 🚫 **Ignores** `.git`, `.venv`, `node_modules`, `__pycache__`, and other common noise.
- 📊 **Counts files by extension** and reports their total size.
- 🧭 **Detects programming languages** from file extensions.
- 📝 **Generates a Markdown report** summarizing the project.
- ⏳ **Shows scan progress** on terminals (auto-disabled when piped or with `--quiet`).
- 🛡️ **Handles errors gracefully** — unreadable files become warnings, not crashes.
- 🧩 **Extensible** — add new metrics or output formats via small, registered plugins.

```bash

pip install -e .

# Or, once published
pip install ai-workspace-assistant
```

This installs the `aiwa` command.

## Usage

```bash
# Analyze the current directory
aiwa

# Analyze a specific project
aiwa ~/code/my-project

# Write the report to a file instead of stdout
aiwa ~/code/my-project -o report.md

# Turn off the progress indicator
aiwa ~/code/my-project --quiet

# Add extra directories to ignore
aiwa ~/code/my-project --ignore dist coverage

# Disable the built-in ignore list entirely
aiwa ~/code/my-project --no-default-ignores

# Show the version
aiwa --version
```

### Options

| Option | Description |
| --- | --- |
| `path` | Project directory to scan (default: current directory). |
| `-o`, `--output FILE` | Write the report to `FILE` instead of stdout. |
| `-f`, `--format FMT` | Report format (default: `markdown`). |
| `-q`, `--quiet` | Suppress progress output. |
| `--ignore DIR [DIR ...]` | Additional directory names to ignore. |
| `--no-default-ignores` | Disable the built-in ignore list. |
| `--version` | Print the version and exit. |

## Architecture

The package is layered so each concern lives in one place:

```
CLI (argparse)  ──▶  Orchestration (main)  ──▶  Scanner  ──▶  Analyzers  ──▶  Reporters
                        │                         │
                        └── Config (ignore rules, language map)
                        └── Progress (TTY / quiet)
                        └── Errors (graceful failures)
```

| Module | Responsibility |
| --- | --- |
| `cli.py` | Parses arguments, wires the pieces together, returns an exit code. |
| `config.py` | Built-in ignore list and the extension → language map. |
| `errors.py` | Custom `ScanError` / `ReportError` exception types. |
| `models.py` | Typed dataclasses: `FileEntry`, `ScanResult`, `ProjectReport`, … |
| `scanner.py` | Walks the directory tree, applies ignore rules, emits a `ScanResult`. |
| `progress.py` | `Progress` protocol + `TtyProgress` / `NullProgress` implementations. |
| `analyzers/` | Turn a `ScanResult` into statistics (extensions, languages). |
| `reporters/` | Turn a `ProjectReport` into output (Markdown). |

The scanner only does filesystem work. *Meaning* is added later by analyzers, and
*presentation* by reporters. The CLI orchestrates the three.

## Extending the project

Two extension points make adding features trivial.

### Add a new analyzer

Create a class that satisfies the `Analyzer` protocol (a `name` and an `analyze`
method) and register it:

```python
# src/ai_workspace_assistant/analyzers/todos.py
from ai_workspace_assistant.config import TODO_ANALYZER_NAME  # add to config.py
from ai_workspace_assistant.models import ScanResult

class TodoAnalyzer:
    name = TODO_ANALYZER_NAME

    def analyze(self, scan: ScanResult) -> int:
        return sum(
            1
            for entry in scan.files
            if entry.extension == ".py"
            and "TODO" in (scan.root / entry.path).read_text(encoding="utf-8")
        )
```

```python
# in analyzers/__init__.py
from ai_workspace_assistant.analyzers.todos import TodoAnalyzer

ANALYZERS.append(TodoAnalyzer())
```

### Add a new reporter

Implement the `Reporter` protocol and register it; `--format` picks it up automatically:

```python
# src/ai_workspace_assistant/reporters/json.py
import json
from ai_workspace_assistant.models import ProjectReport

class JsonReporter:
    format = "json"

    def render(self, report: ProjectReport) -> str:
        return json.dumps(
            {
                "root": str(report.scan.root),
                "total_files": report.scan.total_files,
                "languages": [vars(l) for l in report.get_languages()],
            },
            indent=2,
        )
```

```python
# in reporters/__init__.py
from ai_workspace_assistant.reporters.json import JsonReporter

REPORTERS.append(JsonReporter())
```
